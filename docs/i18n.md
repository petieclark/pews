# Internationalization (i18n) Support

Pews now supports multiple languages to serve churches around the world.

## Supported Languages

- **English (en)** - Default
- **Spanish (es)** - Español
- **Portuguese (pt)** - Português
- **Korean (ko)** - 한국어

## Backend API

### Get Translations

```
GET /api/i18n/:locale
```

Returns all translation strings for the specified locale.

**Example:**
```bash
curl http://localhost:8190/api/i18n/es
```

**Response:**
```json
{
  "nav.dashboard": "Panel",
  "nav.people": "Personas",
  "nav.giving": "Ofrendas",
  ...
}
```

### Get Supported Locales

```
GET /api/i18n/locales
```

Returns a list of all supported locales.

**Example:**
```bash
curl http://localhost:8190/api/i18n/locales
```

**Response:**
```json
{
  "locales": ["en", "es", "pt", "ko"]
}
```

## Frontend Usage

### Translation Store

Import the translation helper in your Svelte components:

```javascript
import { t } from '$lib/i18n.js';

let translate;
const unsubscribe = t.subscribe(value => {
    translate = value;
});
```

Use in your template:

```svelte
{#if translate}
    <h1>{translate('nav.dashboard')}</h1>
    <button>{translate('common.save')}</button>
{/if}
```

### Language Selector Component

Add the language selector to any page:

```svelte
import LanguageSelector from '$lib/LanguageSelector.svelte';

<LanguageSelector />
```

### Manual Language Change

```javascript
import { loadTranslations } from '$lib/i18n.js';

await loadTranslations('es');
```

## Tenant Configuration

Each tenant can set a default locale in their settings:

1. Go to **Settings**
2. Select **Language** from the dropdown
3. Click **Save Changes**

The `default_locale` field is stored in the `tenants` table.

## Adding New Languages

To add a new language:

1. Create a new JSON file in `internal/i18n/locales/` (e.g., `fr.json`)
2. Add all translation keys with French translations
3. Update `internal/i18n/service.go` to include the new locale in the `locales` array
4. Add the locale name to `web/src/lib/i18n.js` in the `localeNames` object

## RTL Support

The foundation for RTL (Right-to-Left) languages like Arabic and Hebrew is in place:

- The `dir` attribute is automatically set on the HTML element
- CSS logical properties should be used for layouts
- To add RTL support, add the locale to the `rtlLocales` array in `web/src/lib/i18n.js`

## Translation Coverage

Current translations cover approximately 150 strings per locale:

- Navigation items
- Common actions (save, cancel, delete, etc.)
- Form labels
- Authentication flows
- Settings pages
- Error messages
- Page titles

## Database Migration

Migration `012_i18n.sql` adds the `default_locale` column to the `tenants` table:

```sql
ALTER TABLE tenants ADD COLUMN default_locale VARCHAR(5) DEFAULT 'en' NOT NULL;
```

## Storage

- **Backend translations:** Embedded in the binary from `internal/i18n/locales/`
- **User preference:** Stored in browser `localStorage`
- **Tenant default:** Stored in `tenants.default_locale` column

## Performance

- Translation files are embedded in the Go binary (no disk I/O at runtime)
- Translations are loaded once on first access and cached
- API responses are cached for 1 hour via `Cache-Control` header
- Frontend stores translations in Svelte store (reactive updates)
